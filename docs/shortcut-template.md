# Shortcut Documentation Template

Use this template when adding a new shortcut to the repository. Copy this structure into `shortcuts/<category>/<shortcut-name>/README.md`.

---

```markdown
# Shortcut Name

One-line description of what the shortcut does.

## Why It Exists

What problem does this solve? Why would someone install this?

## User-Facing Behavior

Numbered list of what the user experiences from tap to result.

## Internal Flow

Step-by-step description or diagram of what the shortcut does internally.

## Inputs

| Input | Description |
|-------|-------------|
| ... | ... |

## Outputs

| Output | Description |
|--------|-------------|
| ... | ... |

## Permissions Required

List every permission the shortcut needs and why.

## Setup

Step-by-step instructions to install and configure.

## Configuration Options

| Option | Default | Description |
|--------|---------|-------------|
| ... | ... | ... |

## Privacy Notes

- What data leaves the device?
- Where does it go?
- What is stored locally?

## Known Limitations

Bullet list of things the shortcut cannot do or edge cases to be aware of.

## Troubleshooting

| Problem | Solution |
|---------|----------|
| ... | ... |
```
